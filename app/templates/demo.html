Option A — TCS Managed
Centralised

TCS admin manages all organisations. Invites users, assigns roles, runs bulk migration across every org from one place.

1
TCS opens migration dashboard — sees all orgs
2
Invites user to any org — email sent via Auth0
3
User accepts → sets password → pending role queue
4
TCS assigns role → user is active
TCS Admin Login
Option B — Org Self-Service
Delegated

Each organisation admin manages their own team. Invites users, assigns roles — scoped to their org only. TCS never involved.

1
Org Admin logs in — sees only their org
2
Invites user to their org — email sent via Auth0
3
User accepts → sets password → pending role queue
4
Org Admin assigns role → user is active

Social

Users sign in with Google, Microsoft, or GitHub. Auth0 normalises identity — platform DB resolves role from the sub claim regardless of provider.

1

2

3

4
