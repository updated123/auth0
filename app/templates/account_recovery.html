Auth0 handles all credential management. The platform backend never stores passwords or manages login credentials directly. All recovery flows below call Auth0's Management API — the platform merely orchestrates the request.
Forgot Password
Live

Already built. User enters their email address and Auth0 sends a secure, one-time password reset link. The link expires after 5 minutes — no password is ever stored or transmitted to the platform backend.

  • Auth0 validates the email is registered
  • Sends a signed, expiring reset URL
  • User sets a new password directly in Auth0
  • Platform DB is never involved

Resend Verification Email
Demo

If a user never received or lost their email verification link, an admin can trigger a resend via Auth0's Management API. The user receives a fresh verification link at their registered address.

Locked Account Recovery
Demo

Auth0 blocks user accounts after too many failed login attempts. A platform admin can unblock the user via the Management API — no password reset required. The user can then log in normally.

Change Email Address
Admin Only

Email changes are handled via Auth0's Management API by platform admins. The user cannot change their own email — it must be done by an admin to prevent account hijacking.

  • Admin calls PATCH /api/v2/users/{'{user_id}'}
  • Auth0 sends a verification email to the new address
  • Old email remains active until the new one is verified
  • Platform DB email column updated on next login via JWT
In this POC, email changes are managed from the Auth0 dashboard. A full implementation would add an admin UI form here that calls PATCH /api/v2/users/{'{user_id}'}.
